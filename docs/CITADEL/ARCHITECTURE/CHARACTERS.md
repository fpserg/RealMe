# Character Library

## Goal

Establish canonical visual identities for recurring RealMe characters before creating large multi-character illustrations.

Future scenes should reuse existing characters rather than redesign them every time.

---

## Principle

Scenes are composed from characters.

Characters are **not** created from scenes.

Character continuity has higher priority than composition, lighting or location.

---

## Workflow

### Phase I — Character Assets

Create one canonical portrait for every Guild member.

- Steward
- Voice
- Observer
- Warden
- Architect
- Librarian

Each portrait becomes the visual reference for all future illustrations.

---

### Phase II — Character Bible

For every character document:

- role
- personality
- appearance
- recurring clothing
- recurring objects
- preferred lighting
- facial expressions
- prohibited visual changes
- canonical prompt

---

### Phase III — Scene Library

After all character assets exist:

- Fireside II
- Guild Council
- Stronghold Visit
- Citadel Library
- Tower Observatory
- Third Realm journeys
- future chronicles

All scenes reuse the canonical characters.

---

## Design Philosophy

Build visual continuity exactly the same way RealMe builds itself.

Stone by stone.

Character by character.

Only then...

Story by story.
