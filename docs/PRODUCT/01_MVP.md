# RealMe MVP

**Status:** Active  
**Version:** MVP v1.0  
**Target:** First working prototype

---

# Purpose

The purpose of the MVP is to validate a single hypothesis:

> A continuously maintained World Model can reduce cognitive load through natural conversation while preserving human agency.

The MVP is successful if the user naturally offloads remembering, organizing and tracking to RealMe without changing how they communicate.

---

# Product Vision

RealMe is a conversation-first cognitive operating system.

It continuously builds and maintains an understanding of the Warden's operating environment.

The user speaks naturally.

RealMe maintains the structure.

Over time, the accumulated understanding enables increasingly useful assistance without requiring the user to manually organize information.

---

# MVP Scope

The MVP should support four fundamental capabilities.

## 1. Understand Conversation

The user interacts exclusively through natural language.

There are no forms, manual databases or structured editors.

Conversation is the only required interface.

---

## 2. Build And Maintain A World Model

Every meaningful conversation may update the persistent World Model.

The World Model stores information required to understand the Warden's operating environment across time.

It includes:

- reality;
- history;
- current state;
- commitments.

The user never edits the World Model directly.

---

## 3. Use The World Model During Reasoning

Future conversations should use accumulated understanding.

RealMe should not rely only on the current conversation.

Examples:

- remembering ongoing projects;
- understanding people and relationships;
- recognizing recurring routines;
- recalling previous decisions;
- understanding deadlines and commitments.

---

## 4. Reduce Cognitive Load

The World Model should enable RealMe to assist by reducing unnecessary cognitive effort.

Examples include:

- remembering commitments;
- surfacing relevant context;
- identifying forgotten obligations;
- recognizing recurring activities;
- connecting related information across conversations.

The MVP demonstrates that accumulated understanding improves assistance over time.

---

# What The MVP Is Not

The MVP is not intended to become a complete productivity platform.

It is not:

- a task manager;
- a calendar application;
- a note-taking application;
- a personal knowledge base;
- an autonomous AI agent.

These capabilities may emerge later if they support the mission, but they are not objectives of the MVP.

---

# User Experience

The expected interaction is simple.

The user lives normally.

The user talks to RealMe.

RealMe continuously updates its understanding.

That understanding improves future conversations.

The user should experience progressively less need to remember, organize or reconstruct context.

---

# Example

### Wednesday Morning

> Sergey Jr has football today at 18:00.

RealMe recognizes an existing recurring commitment.

Later that day:

> Don't forget Sergey Jr's football at 18:00. You also planned to finish the Equity Strategy outline today.

The reminder is generated from the World Model rather than today's conversation.

---

### Evening

> We finished football. The Equity Strategy outline is about halfway complete.

The World Model is updated.

Future conversations now understand the current project state.

---

# Success Criteria

The MVP is successful if it demonstrates that:

- natural conversation is sufficient to maintain the World Model;
- the World Model improves future conversations;
- commitments persist across sessions;
- context no longer needs to be repeatedly explained;
- assistance meaningfully reduces cognitive load.

---

# Explicitly Out Of Scope

The MVP does not attempt to solve:

- autonomous decision-making;
- autonomous prioritization;
- automatic execution of work;
- complex workflow automation;
- multi-user collaboration;
- optimization algorithms;
- advanced analytics;
- integrations with external services.

---

# Guiding Principle

Every MVP feature should answer one question:

> Does this help reduce cognitive load while preserving human agency?

If the answer is no, the feature does not belong in the MVP.
