## Project Vision

RealMe is a conversation-first cognitive operating system.

The user interacts only through natural conversation.

The World Model is the persistent representation of the user's personal reality.

The LLM is the reasoning engine.

Conversation updates the World Model through Living Input.

The user never maintains the World Model directly.

The goal is to reduce cognitive load rather than create another productivity tool.

## Repository Philosophy

Repository structure is organized by architectural responsibility rather than framework.

Conversation is the interface.

The World Model is the center of the architecture.

The LLM is replaceable.

The World Model must never depend on the LLM.

## MVP Goal

The July 31 prototype should demonstrate only one architectural hypothesis:

> Conversation is the interface.
> The World Model is the memory.

A successful MVP allows the user to:

- converse naturally,
- submit Living Inputs,
- automatically update the World Model,
- use the updated World Model in future conversations.

Nothing beyond this is required for MVP.
