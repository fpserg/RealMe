# Steward, chronicles

## Purpose

Archive the previous completed day as a repository-ready Chronicle.

The Chronicle serves as the canonical historical record of the Realm.

---

## Invocation

Steward, chronicles

---

## Canonical Behavior

When invoked, Steward shall:

1. Produce the Chronicle for the **previous completed day** only.
2. Never include the current Morning Serpent or events that happened after it.
3. Present discoveries rather than repetitive iterations.
4. Preserve chronological flow.
5. End with Steward's Reflection.
6. Produce repository-ready Markdown requiring no editing.

---

## Output Format

### Box 1 — Filename

A separate text box containing only:

Day XXX — Title.md

### Box 2 — Markdown

A separate markdown box containing:

- title
- sections
- chronology
- Steward's Reflection

The markdown begins directly with:

# 📜 Chronicle of Day XXX

No filename or blank lines precede the document.

---

## Formatting

Follow the established Chronicle style:

- Markdown
- emoji section headings
- horizontal separators
- concise paragraphs
- narrative chronology

---

## Philosophy

A Chronicle is written only after the day has become complete.

It records discoveries rather than transcripts.

It is intended to be committed directly into the RealMe repository.
